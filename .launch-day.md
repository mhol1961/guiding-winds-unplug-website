### 0930 — DNS cutover

If the current registrar is **not** Cloudflare:

- [ ] Log in to current registrar.
- [ ] Change name servers to Cloudflare's: `{xxx}.ns.cloudflare.com` and `{yyy}.ns.cloudflare.com` (per the project's Cloudflare DNS dashboard).
- [ ] Wait for propagation (5–60 minutes). Check with `dig guidingwinds-unplug.com NS +short`.

If DNS is already on Cloudflare:

- [ ] In Cloudflare dashboard → DNS → Records, set the apex `A` and `www` `CNAME` to point to Cloudflare Pages: `apex` → `CNAME guidingwinds-unplug.pages.dev` (proxied), `www` → `CNAME guidingwinds-unplug.pages.dev` (proxied).
- [ ] Set `www` → apex redirect via Cloudflare Page Rules: `https://www.guidingwinds-unplug.com/*` → `https://guidingwinds-unplug.com/$1` (301).
- [ ] Verify SSL is set to **Full (Strict)** under SSL/TLS.
- [ ] Enable **HSTS** with `max-age=15552000` (180 days) and `includeSubDomains`. Skip `preload` until v1 has been stable for 90 days.

## Rollback procedure

If anything goes seriously wrong in the first 24 hours:

1. **DNS revert.** In Cloudflare DNS, revert apex `A` and `www` `CNAME` to the old values (have these screenshotted before cutover).
2. **Pages deployment revert.** Cloudflare Pages → Deployments → roll back to the previous deploy.
3. **Email Dodie + Clint:** notify them the site is temporarily back on the old version while we fix.
4. **Diagnose** in a fresh Claude Code session against the failing route. Don't fix in production.

Common quick-fixes (no rollback needed):

| Symptom | Quick fix |
|---|---|
| Inquiry form posts but contact doesn't land in GHL | Re-verify `GHL_PRIVATE_TOKEN` in Cloudflare Pages env vars. Token expires if not used for 6+ months. |
| Calendar shows wrong cabin counts | Edit the voyage `.md` files in `src/content/voyages/` and push; rebuild auto-triggers. Don't try to live-edit. |
| 5xx on /api/* | Check Cloudflare Pages Functions logs. Most likely env var issue or GHL token expiry. |
| Lighthouse drops below targets | Check if Dodie added a large image to /public — if so, run it through Squoosh and re-add. |
